Print Odoo Reports via Zebra Printer
Odoo Extension Allowing The User To Print Reports Directly Via Zebra Printer Instead Of Downloding As A PDF File.

This module uses the QZ Tray client for sending direct print commands to the printer:

  • Installation Link: ( https://qz.io/download/ )
  • Demo Link: ( https://demo.qz.io/ )

Make sure you have installed the python library zplgrf.

  • Installation Command: ( pip install zplgrf )

Salient Features

  • User can select the default report action for each report i.e whether to download the report or to send it to the printer.
  • User can select the default Zebra printer for a report.
  • User can select the desired printer from the list of available printers while printing the report.
  • User can also define custom report templates using his printer specific language(like zpl) if he doesn't wish to print the default Odoo's report.

Setup & Configuration

Configuring Printers & Custom Report Templates

You can setup the available printers and custom report templates under the Settings > Technical menu.

Make sure the Printer's name is exactly the same as the name of the printer is used by the QZ Tray client to make connection and send print commands.

Printer type : Select the printer type here. Currently, only zebra printers are supported.

Custom Report Templates : User can also define custom report templates using their printer specific language(like ZPL) if he doesn't wish to print the default Odoo's report.

Odoo-Report Configuration.

Next, you'll need to configure the report action in the report that you wish to send to the printer. You can set this from the Settings > Technical menu under the Reporting section.

In the desired report form view, you can set the following parameters:

  • Report Action : Select the "Send To Printer" option here if you wish to print the report directly instead of downloading it. The default value of this field is "Default Odoo's Action" that leads to the default Odoo's behaviour.
  • Printer : Select the default printer here. Leave this field blank if you wish to select the printer from the list of available printers while printing it.
  • Use Template : Select this option if you wish to use a custom Report Template for printingthis report. If you leave this filed blank then the default Odoo's pdf report will be converted into the the printer specific commands (ZPL) and then sent for printing.
  • Report Template : Select the desired Report Template record here if the Use Template filed is marked.

Work-flow

When Default Printer Is Set

When the default printer is set in the report configuration then the report is sent to that printer directly when the 'Print' server action is triggered. This can be used in the cases when the user uses the same printer always for a report.

When Default Printer Is Not Set

In case when no default printer is set in the report configuration, then a popup appears with the list of all the available printers and the user can select the desired one while printing the same.
